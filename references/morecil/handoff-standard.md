# Morecil Handoff Standard

Используй этот формат, когда одна роль передаёт работу следующей.

## Короткий handoff

- Роль-отправитель
- Роль-получатель
- Что готово
- Что осталось
- Какие решения заморожены
- Какие риски остались
- Какие файлы нужно читать следующими

## Пример хорошего handoff

- Роль-отправитель: `solution-architect`
- Роль-получатель: `frontend-builder`
- Что готово: утверждён маршрут, выбран стек, зафиксирован MVP и список состояний интерфейса
- Что осталось: реализовать основной UI-маршрут и критичные состояния
- Какие решения заморожены: Next.js, custom visual layer, без auth в первой версии
- Какие риски остались: форма отправки пока без реального provider
- Какие файлы нужно читать следующими: `implementation-plan.md`, `design-direction.md`, `active-context.md`

## Что нельзя писать в handoff

- “всё понятно из контекста”
- “просто доделай”
- длинную простыню без приоритетов
- новые архитектурные идеи, которые не были согласованы
