# Morecil Delegated Orchestration

Это расширение `Morecil Role System` для случаев, когда проект достаточно сложный, чтобы часть ролей работала как отдельные под-агенты.

## Когда включать delegated mode

- Есть хотя бы две независимые подсистемы.
- Фронтенд, бэкенд, data или automation можно вести параллельно.
- Оркестратор может продолжать локальную работу, пока под-агент делает bounded subtask.

## Когда не включать

- Discovery ещё не завершён.
- План не утверждён.
- Следующий шаг полностью блокируется одной неизвестной задачей.
- Работа слишком мелкая и делегирование только расходует внимание и токены.

## Как делегировать правильно

- Каждому под-агенту нужен ownership по файлам или ответственности.
- У под-агента должен быть понятный ожидаемый результат.
- У под-агента должен быть короткий handoff обратно оркестратору.
- Оркестратор не должен дублировать уже делегированную работу.

## Хорошие кандидаты для делегирования

- `frontend-builder`
- `backend-builder`
- `database-designer`
- `automation-builder`
- `qa-reviewer`

## Плохие кандидаты для делегирования

- discovery как первичный контакт с пользователем
- финальный handoff без контроля оркестратора
- спорные архитектурные развилки, где ещё не хватает решения пользователя
