# Маршрут реализации: Общий веб-продукт

## Когда использовать

Для веб-продукта, который уже больше, чем лендинг, но ещё не тянет на полноценный SaaS.

## Порядок

1. Зафиксировать первый полезный пользовательский маршрут.
2. Разделить зоны ответственности по слоям.
3. Собрать минимальный рабочий slice.
4. Добавить нужные интеграции и проверки.
5. Подготовить handoff и env.
