# Правила anti-slop

Не выдавай типовой и безликий результат за “готовый продукт”.

## Правила

- не делай generic hero и generic dashboard
- не заполняй проект бессмысленными карточками
- не добавляй фичи “на будущее” без спроса
- не маскируй слабую архитектуру красивыми словами
- качество важнее видимости объема
