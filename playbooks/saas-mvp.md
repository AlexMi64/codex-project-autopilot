# Маршрут реализации: SaaS MVP

## Когда использовать

Для кабинета, dashboard, auth-flow, внутреннего сервиса или MVP-приложения с данными и логикой.

## Порядок

1. Зафиксировать пользовательские сценарии.
2. Разделить frontend, backend и database.
3. Определить auth и ownership.
4. Собрать минимальный рабочий маршрут.
5. Добавить проверки, env и handoff.
