# Маршрут реализации: API-интеграционный воркер

## Когда использовать

Для webhook-сервисов, внешних API-интеграций, синхронизации данных и provider-driven логики.

## Порядок

1. Зафиксировать контракт внешнего API.
2. Добавить валидацию входа и auth.
3. Реализовать основной happy path.
4. Добавить retry, dedup и idempotency.
5. Подготовить verification и handoff.
