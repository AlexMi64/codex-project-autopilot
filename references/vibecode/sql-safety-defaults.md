# SQL-безопасность по умолчанию

Если проект хоть как-то касается базы, API или авторизации, SQL-безопасность не должна оставаться “на потом”.

## Правила

- никакой конкатенации SQL-строк из пользовательского ввода
- использовать только parameterized queries или ORM
- валидировать вход до запроса: Zod, Pydantic, Joi или аналог
- ограничивать попытки там, где есть логин, поиск по чувствительным полям или публичный endpoint
- не выполнять запросы под root/postgres/service_role без реальной причины
- перед handoff отметить, где именно проверялась SQL-безопасность

## Для плагина

- если проекту не нужна база, не добавляй её ради “правильной архитектуры”
- если база нужна, backend и database роли обязаны зафиксировать, как закрыта SQL-инъекция
- в verification нужно явно проверить, что в коде нет ручной конкатенации SQL
