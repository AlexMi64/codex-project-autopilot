# Базовый чеклист безопасности для новичка

Даже новичковый проект должен иметь безопасные дефолты без лишней паранойи.

## Правила

- секреты только через env
- `.env` не хранится в репозитории
- валидация входных данных обязательна
- auth и authorization не одно и то же
- если есть данные пользователей, защита доступа должна быть явной
- финальный checklist по секретам обязателен
