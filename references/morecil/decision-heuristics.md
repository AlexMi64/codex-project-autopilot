# Morecil Decision Heuristics

Это набор эвристик для ролей и оркестратора.

## Scope

- Один рабочий сценарий лучше трёх полуготовых.
- Рекомендации живут отдельно от MVP.
- Если решение добавляет новую подсистему, нужно отдельное обоснование.

## UX

- Понятность важнее количества экранов.
- Сильный характер интерфейса важнее generic-красоты.
- У каждого критичного действия должно быть понятное состояние: loading, success, error, empty.

## Backend

- Не прячь сбои.
- Логи и валидация обязательны там, где есть входные данные или внешние сервисы.
- Интеграция без failure-path — недоделанная интеграция.

## Data

- База не нужна, пока не доказано обратное.
- Ownership и доступ важнее количества таблиц.
- Durable state и transient state должны быть разделены.

## Handoff

- Следующая роль не должна догадываться, что имелось в виду.
- Источник истины должен быть назван явно.
- Если решение спорное, оно должно быть отмечено как спорное.
